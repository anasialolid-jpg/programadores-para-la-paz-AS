Al enviar una petición POST a http://localhost:3000/reportes, el servidor captura las propiedades "tipo" y "descripcion", les asigna un identificador numérico único (id: 1) y almacena el objeto estructurado dentro de un arreglo en la memoria RAM del servidor.

Posteriormente, al realizar una consulta mediante la ruta GET a http://localhost:3000/reportes, el servidor responde con un código de estado HTTP 200 y devuelve un arreglo JSON que contiene la lista con todos los reportes registrados secuencialmente. 

La respuesta esperada del GET es:
[
  {
    "id": 1,
    "tipo": "Infraestructura",
    "descripcion": "Daño en alumbrado público"
  }
]
